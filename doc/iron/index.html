The main crate for the Iron library.

Iron is a high level web framework built in and for Rust.

Iron does not come bundled with any middleware - instead, Iron provides a robust and efficient framework for creating and plugging in middleware.

Obligatory example:

#[deriving(Clone)]
struct ResponseTime {
    entry_time: u64
}

impl ResponseTime { fn new() -> ResponseTime { ResponseTime { entry_time: 0u64 } } }

impl Middleware for ResponseTime {
    fn enter(&mut self, _req: &mut Request, _res: &mut Response) -> Status {
        self.entry_time = precise_time_ns();
        Continue
    }

    fn exit(&mut self, _req: &mut Request, _res: &mut Response) -> Status {
        let delta = precise_time_ns() - self.entry_time;
        println!("Request took: {} ms", (delta as f64) / 1000000.0);
        Continue
    }
}

// ...
server.chain.link(ResponseTime::new());
// ...

Enums

Status

The Status returned by Middleware's enter and exit methods. This indicates to the Chain whether this is a terminal Middleware or whether to continue passing the Request and Response down the Chain's stack.

Traits

Chain

Chains are the backbone of Iron. They coordinate Middleware to ensure they are resolved and called in the right order.
Middleware

All middleware should implement Middleware, which allows it to be linked to a Chain so that it will be called for each incoming request.
Plugin

An interface for plugins that cache values between calls.
